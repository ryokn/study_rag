## 実装アイデア

### A. RAG精度向上

- [ ] **HyDE（Hypothetical Document Embeddings）**: 質問から仮想回答を生成してベクトル検索することで関連チャンクの取得精度を上げる
- [ ] **リランキング**: 検索結果を別モデルで再スコアリングし、上位チャンクの精度を向上させる
- [ ] **クエリ拡張**: 質問を複数の言い回しに展開してから検索することで網羅性を高める

### B. エージェント拡張

- [ ] **並列エージェント**: マルチエージェントの ResearchAgent を複数同時実行し、結果を統合する
- [ ] **専門エージェント分割**: PDF専用・Web専用など用途別にエージェントを分け、Supervisor が振り分ける
- [ ] **エージェント評価**: エージェントの各ツール呼び出し精度・コスト・レイテンシを MLflow で記録する

### C. データパイプライン

- [ ] **Delta Lake 連携**: ChromaDB の代わりに Delta Lake をベクトルストアとして試す
- [ ] **ストリーミング取り込み**: PDFを追加するたびに自動でベクトル化・差分更新する仕組み
- [ ] **Web スクレイピング**: URLを指定してWebページをPDF同様に取り込めるようにする

### D. 評価・モニタリング

- [ ] **コンテキスト精度（context_precision）の追加**: RAGAS の追加メトリクスを評価対象に含める
- [ ] **A/Bテスト自動化**: チャンクサイズ・top-k のグリッドサーチを自動で回して MLflow に記録する
- [ ] **レイテンシ計測**: 各ノード・ツールの実行時間をトレースして可視化する

### E. UI・UX

- [ ] **Streamlit への multi-agent モード追加**: Web UI からも HITL フローを操作できるようにする
- [ ] **チャット履歴の永続化**: ブラウザを閉じても会話履歴が残るようにする（SQLite等）
